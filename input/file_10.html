Vitest的配置选项详解

详细介绍Vitest的各种配置选项及其用途

tune

基础配置

Vitest的基础配置选项，用于控制测试文件匹配、环境设置等

export default defineConfig({
  test: {
    // 测试文件匹配模式
    include: ['**/*.{test,spec}.{js,ts,jsx,tsx}'],
    // 排除的文件或目录
    exclude: ['node_modules/**', 'dist/**'],
    // 测试环境
    environment: 'jsdom',
    // 全局API启用
    globals: true
  }
})
check_circle
include/exclude - 控制哪些文件被视为测试文件
check_circle
environment - 设置测试运行环境（node/jsdom/happy-dom）
speed

性能优化

优化测试执行性能的配置选项，包括并行测试、缓存策略等

export default defineConfig({
  test: {
    // 测试池类型
    pool: 'threads', // 'threads' | 'forks' | 'vmThreads'
    // 最大并发数
    maxConcurrency: 5,
    // 测试超时时间(ms)
    testTimeout: 5000,
    // 钩子超时时间(ms)
    hookTimeout: 10000
  }
})
check_circle
pool - 控制测试运行方式，提高并行执行效率
check_circle
maxConcurrency - 限制同时运行的测试数量
analytics

覆盖率配置

配置代码覆盖率报告的选项，帮助评估测试完整性

export default defineConfig({
  test: {
    coverage: {
      // 报告类型
      reporter: ['text', 'json', 'html'],
      // 包含的文件
      include: ['src/**/*'],
      // 排除的文件
      exclude: ['node_modules/**'],
      // 覆盖率阈值
      thresholds: {
        global: {
          lines: 80,
          functions: 80
        }
      }
    }
  }
})
check_circle
reporter - 设置覆盖率报告格式
check_circle
thresholds - 定义覆盖率最低要求
build

高级配置

Vitest的高级配置选项，包括设置文件、报告器、别名等

export default defineConfig({
  test: {
    // 测试前运行的设置文件
    setupFiles: ['./test-setup.js'],
    // 测试报告器
    reporters: ['verbose', 'json'],
    // 模拟设置
    mockReset: true,
    // 别名配置
    alias: {
      '@': './src'
    },
    // 全局变量
    define: {
      __APP_VERSION__: '"1.0.0"'
    }
  }
})
check_circle
setupFiles - 在测试前运行的设置文件
check_circle
alias - 配置模块导入别名
