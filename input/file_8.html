Vitest的API介绍

详细介绍Vitest的核心API，如test、describe、expect等

science

test / it

定义一个测试用例，test和it功能相同，是Vitest中最基础的API

import { test, expect } from 'vitest'

// 基本用法
test('adds 1 + 2 to equal 3', () => {
  expect(1 + 2).toBe(3);
});

// 使用it别名
it('multiplies 2 * 3 to equal 6', () => {
  expect(2 * 3).toBe(6);
});
异步测试 超时设置 跳过测试
category

describe

将相关测试分组到测试套件中，提高测试组织性和可读性

import { describe, test, expect } from 'vitest'

describe('Math functions', () => {
  describe('add()', () => {
    test('adds positive numbers', () => {
      expect(add(1, 2)).toBe(3);
    });

    test('adds negative numbers', () => {
      expect(add(-1, -2)).toBe(-3);
    });
  });
});
嵌套分组 跳过套件 只运行套件
rule

expect

用于创建断言，验证代码行为是否符合预期，提供丰富的匹配器

import { expect } from 'vitest'

// 基本断言
expect(value).toBe(expected);
expect(value).toEqual(expected);
expect(value).toBeTruthy();
expect(value).toBeFalsy();

// 数组和对象
expect(array).toContain(item);
expect(obj).toHaveProperty(key);

// 异常
expect(() => fn()).toThrow();
否定断言 异步断言 自定义匹配器
build

生命周期钩子

提供beforeEach、afterEach、beforeAll和afterAll等钩子函数，用于测试设置和清理

import { beforeEach, afterEach } from 'vitest'

beforeEach(() => {
  // 每个测试前运行
  initializeDatabase();
});

afterEach(() => {
  // 每个测试后运行
  clearDatabase();
});

beforeAll(() => {
  // 所有测试前运行一次
  startServer();
});
异步钩子 作用域控制 资源管理
